# Security Policy

## 支持范围

当前项目处于个人本地工具阶段，优先修复以下安全问题：

- API key、环境变量或本地敏感数据泄露风险。
- 文件上传、CSV 解析或本地缓存导致的异常行为。
- 后端接口暴露导致的非预期本地数据读取或写入。
- AI 请求中意外携带不必要隐私数据。

## 敏感信息

请不要提交或公开：

- `.env`
- `AI_API_KEY`
- 真实持仓数据
- 本地导入 CSV
- `data/cache/`
- `data/results/`
- 运行日志

如果误提交了 API key，请立即在服务商控制台吊销并重新生成。

## 报告问题

请通过 GitHub issue 或私下联系维护者报告安全问题。报告中请尽量包含：

- 影响范围
- 复现步骤
- 可能泄露的数据类型
- 建议修复方向

请不要在公开 issue 中粘贴真实 API key、持仓明细或完整缓存文件。
